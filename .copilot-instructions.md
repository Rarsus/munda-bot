## Core Principles

### 1. **Data Segregation (HARD REQUIREMENT)**
- User data and guild data MUST be completely separate
- Guild members segregated by guild_id
- Users can ONLY access/modify their own data
- Guild owners can ONLY manage their guild's data

```typescript
// ❌ ANTI-PATTERN: Mixing user and guild data
const userData = { id, username, guildInfo, guildRoles };

// ✅ PATTERN: Segregated by design
const userGlobal = await userRepository.getUser(userId);
const guildMember = await memberRepository.getMember(guildId, userId);
```

### 2. **GDPR Compliance First**
- All data processing requires explicit consent
- Every data access must be audited
- Users must be able to retrieve, modify, and delete their own data
- Audit logs NEVER deleted (3-year legal retention)

### 3. **Type Safety**
- All functions must have full type definitions
- No `any` types unless absolutely necessary (with comment explaining why)
- All database operations return typed interfaces
- Compilation must pass with `npm run build`

### 4. **Error Handling**
- Custom error classes for all error types (AppError, ValidationError, AuthorizationError)
- Never expose internal errors to users
- Log errors with context but sanitize messages for client

### 5. **Immutability in Audit Trail**
- Audit logs use soft deletes only (mark deleted_at, never actually delete)
- Retention dates enforced by database constraints
- Change history tracked with before/after values

## Architecture Patterns

### Command Pattern (for Bot Commands)
All commands must extend the `Command` base class:

```typescript
import { Command } from '../base';

export class MyCommand extends Command {
  name = 'mycommand';
  description = 'What this command does';
  aliases = ['alias1', 'alias2'];
  usage = 'mycommand [args]';
  examples = ['mycommand arg1', 'mycommand arg2'];
  requiredPermissions = ['SEND_MESSAGES'];

  async execute(context: Message | CommandInteraction): Promise<void> {
    // Implementation
  }
}
```

### Repository Pattern (for Data Access)
All data access through repositories extending `BaseRepository<T>`:

```typescript
export class MyRepository extends BaseRepository<IMyEntity> {
  constructor(pool: Pool) {
    super(pool, 'my_table_name');
  }

  async getById(id: string): Promise<IMyEntity | null> {
    const result = await this.query(
      `SELECT * FROM ${this.tableName} WHERE id = $1`,
      [id]
    );
    return result.rows[0] || null;
  }
}
```

### Middleware Pattern (for Cross-Cutting Concerns)
Middleware functions handle auth, GDPR, logging:

```typescript
// Enforce permission check
await requirePermissions(context, ['ADMINISTRATOR']);

// Enforce consent
await requireConsent(userId, ConsentType.ANALYTICS, gdprService);

// Verify owner
const hasAccess = await verifyDataOwnership(requestingUserId, dataOwnerUserId);
```

### Service Pattern (for Business Logic)
Services orchestrate repositories and middleware:

```typescript
export class MyService {
  constructor(
    private userRepository: UserRepository,
    private auditRepository: AuditRepository,
    private gdprService: GDPRService
  ) {}

  async performAction(userId: string, actionData: any): Promise<void> {
    // Verify user owns data
    await verifyDataOwnership(userId, userId);

    // Check consent
    if (!await gdprService.hasConsent(userId, ConsentType.DATA_COLLECTION)) {
      throw new AuthorizationError('User must consent to data collection');
    }

    // Perform action
    //...

    // Audit log
    await auditRepository.logEvent({
      event_type: 'ACTION_PERFORMED',
      subject_user_id: userId,
      action: 'User performed action',
    });
  }
}
```

## GDPR Compliance Requirements

**These apply to ALL features accessing user/guild data:**

### Before Processing Data
```typescript
// 1. Verify user owns the data
const owns = await verifyDataOwnership(requestingUserId, dataSubjectUserId);
if (!owns) throw new AuthorizationError('No access');

// 2. Check consent exists
const hasConsent = await gdprService.hasConsent(userId, consentType);
if (!hasConsent) throw new AuthorizationError('Consent required');

// 3. Process the data
const data = await repository.getData(dataId);

// 4. Log the access
await logDataAccess(gdprService, userId, 'user', userId, 'Accessed own data');
```

### When Storing Data
```typescript
// Only collect necessary data
const user = {
  user_id: discordUser.id,           // ✅ Necessary
  username: discordUser.username,     // ✅ Necessary
  email: discordUser.email,           // ❌ Avoid unless consented
  ipAddress: request.ip,              // ❌ Never store directly
  lastPageViewed: 'some-page',        // ❌ Unnecessary tracking
};
```

### When Deleting Data
```typescript
// Use repository methods for atomic deletion
const client = await pool.connect();
try {
  await client.query('BEGIN');
  
  // Delete user data
  await userRepository.permanentlyDeleteUser(userId);
  
  // Delete guild memberships
  await memberRepository.deleteUserFromAllGuilds(userId);
  
  // Log erasure
  await auditRepository.logEvent({
    event_type: 'ERASURE_COMPLETED',
    subject_user_id: userId,
    action: 'User data erased',
  });
  
  await client.query('COMMIT');
} catch (err) {
  await client.query('ROLLBACK');
  throw err;
}
```

### Consent Types
Only these 4 consent types exist:

```typescript
enum ConsentType {
  DATA_COLLECTION = 'data_collection',      // Basic bot operation
  MESSAGE_LOGGING = 'message_logging',      // Logging user messages
  ANALYTICS = 'analytics',                   // Usage analytics
  MARKETING = 'marketing',                   // Promotional messages
}
```

Add new types only with GDPR assessment.

## Code Style & Standards

### TypeScript Compilation
- Always run `npm run build` before committing
- Must compile with zero errors
- Use strict mode (tsconfig.json enforces this)

### Naming Conventions

**Files**:
```
PascalCase.ts      - Classes, interfaces, command types
camelCase.ts       - Utility functions, services
SCREAMING_CASE.ts  - Constants (rare)
```

**Classes**:
```typescript
class UserRepository {}      // ✅ Repositories
class MyCommand extends Command {}   // ✅ Commands (ends with: Command)
class GDPRService {}         // ✅ Services
class AppError extends Error {}      // ✅ Errors
interface IUser {}           // ✅ Interfaces (start with: I)
enum ConsentType {}          // ✅ Enums
```

**Variables**:
```typescript
const userId = '...';        // ✅ Camel case
const MAX_RETRIES = 3;       // ✅ Constants are SCREAMING_CASE
let tempValue = null;        // ✅ Camel case
```

**Functions**:
```typescript
async function getUserData() {}       // ✅ Camel case
export async function createUser() {} // ✅ Public exports
```

**Database**:
```
gdpr_user_data          // ✅ Snake case for tables
gdpr_audit_log
user_id, created_at     // ✅ Snake case for columns
```

### Import Organization
```typescript
// 1. External imports (discord.js, pg, etc.)
import { Client, Message, CommandInteraction } from 'discord.js';
import { Pool } from 'pg';

// 2. Internal imports (organized by depth)

// 3. Type imports

```

```typescript

 * Longer description with context:
 * - What it does
 * - When to use it

 */

```

### Comments
```typescript
// ✅ DO: Explain WHY, not WHAT

// ❌ DON'T: Explain obvious code
// Get the user
```

## Database & Data Handling

### Connection Pattern
```typescript
// Use connection pool from database service
import { getDatabase } from './services/database';

const db = getDatabase();
const result = await db.pool.query('SELECT * FROM users');
```

### Repository Methods Always Return Types
```typescript
// ✅ CORRECT: Return types specified
async getUser(userId: string): Promise<IUser | null> {
  const result = await this.query('SELECT * FROM users WHERE id = $1', [userId]);
  return result.rows[0] || null;
}

// ❌ WRONG: Missing return type
async getUser(userId: string) {
  const result = await this.query('...');
  return result.rows[0];
}
```

### Always Use Parameterized Queries
```typescript
// ✅ CORRECT: Parameterized (prevents SQL injection)
const result = await pool.query(
  'SELECT * FROM users WHERE id = $1',
  [userId]
);

// ❌ WRONG: String concatenation (SQL injection vulnerability!)
const result = await pool.query(`SELECT * FROM users WHERE id = '${userId}'`);
```

### Transaction Pattern
```typescript
const client = await this.pool.connect();
try {
  await client.query('BEGIN');
  
  // All operations here are transactional
  await client.query('UPDATE users SET ...');
  await client.query('INSERT INTO audit_log ...');
  
  await client.query('COMMIT');
} catch (error) {
  await client.query('ROLLBACK');
  logger.error('Transaction failed:', error);
  throw error;
} finally {
  client.release();
}
```

### Soft Delete Pattern (for GDPR)
```typescript
// Mark for deletion, don't actually delete
await this.query(
  `UPDATE ${this.tableName} SET deleted_at = NOW() WHERE id = $1`,
  [id]
);

// Query only active records
await this.query(
  `SELECT * FROM ${this.tableName} WHERE deleted_at IS NULL`
);
```

### JSON Storage in PostgreSQL
```typescript
// Storing arrays/objects
await this.query(
  `INSERT INTO table (id, roles) VALUES ($1, $2)`,
  [id, JSON.stringify(roles)] // Store as JSON string
);

// Querying JSON
const result = await this.query(
  `SELECT roles-> #>> '{0}' as first_role FROM table WHERE id = $1`,
  [id]
);
```

## Scenario-Based Guidance

### Scenario 1: Adding a New Command

**File**: `src/commands/[feature]/[FeatureName]Command.ts`

```typescript
import { Message, CommandInteraction, MessageEmbed } from 'discord.js';
import { Command } from '../base';
import { IMyFeature } from '../../interfaces/[feature]/I[Feature]';
import { logger } from '../../services/logger';

export class MyFeatureCommand extends Command {
  name = 'myfeature';
  description = 'What this command does';
  aliases = ['alias'];
  usage = 'myfeature [args]';
  examples = ['myfeature example'];
  requiredPermissions: string[] = [];

  async execute(context: Message | CommandInteraction): Promise<void> {
    // Extract user ID (works for both Message and Interaction)
    const userId = context instanceof Message 
      ? context.author.id 
      : context.user.id;

    try {
      // ALWAYS: Error handling
      
      // 1. Verify ownership/permissions (if applicable)
      const owns = await verifyDataOwnership(userId, targetUserId);
      if (!owns) {
        throw new AuthorizationError('No access');
      }

      // 2. Check consent (if accessing data)
      const hasConsent = await gdprService.hasConsent(userId, ConsentType.DATA_COLLECTION);
      if (!hasConsent) {
        throw new AuthorizationError('Consent required');
      }

      // 3. Perform operation
      const result = await service.performAction(userId, data);

      // 4. Log the action (audit)
      await auditRepository.logEvent({
        event_type: 'ACTION_PERFORMED',
        subject_user_id: userId,
        action: 'User performed action',
      });

      // 5. Send response (embed for richness)
      const embed = new MessageEmbed()
        .setColor('#00ff00')
        .setTitle('✅ Success')
        .setDescription('Action completed')
        .setTimestamp();

      if (context instanceof Message) {
        await context.reply({ embeds: [embed] });
      } else {
        await context.reply({ embeds: [embed], ephemeral: true });
      }
    } catch (error) {
      logger.error(`Error in MyFeatureCommand for user ${userId}:`, error);
      
      const embed = new MessageEmbed()
        .setColor('#ff0000')
        .setTitle('❌ Error')
        .setDescription(error instanceof Error ? error.message : 'Unknown error')
        .setTimestamp();

      if (context instanceof Message) {
        await context.reply({ embeds: [embed] });
      } else {
        await context.reply({ embeds: [embed], ephemeral: true });
      }
    }
  }
}
```

## Common Patterns

### Error Handling Pattern
```typescript
import { AppError, ValidationError, AuthorizationError, NotFoundError } from '../middleware/errorHandler';

// ✅ Use custom error classes
try {
  if (!userId) {
    throw new ValidationError('User ID is required');
  }

  const user = await repository.getById(userId);
  if (!user) {
    throw new NotFoundError(`User ${userId} not found`);
  }

  }

    logger.warn(`Known error: ${error.message}`);
    // Send user-friendly message
  } else {
    logger.error('Unexpected error:', error);
    // Send generic message
  }
}
```

## Anti-Patterns & Gotchas

### ❌ Anti-Pattern: Storing User Data Without Consent
```typescript
// WRONG: Store data without checking consent
async function logMessage(message: Message) {
  await messageRepository.create({
    user_id: message.author.id,
    content: message.content,  // ❌ No consent check!
    timestamp: new Date(),
  });
}

// CORRECT: Check consent first
async function logMessage(message: Message) {
  const hasConsent = await gdprService.hasConsent(
    message.author.id,
    ConsentType.MESSAGE_LOGGING
  );
  
  if (!hasConsent) {
    logger.info(`Message from ${message.author.id} not logged (no consent)`);
    return;  // Skip logging for this user
  }

  await messageRepository.create({
    user_id: message.author.id,
    content: message.content,
    timestamp: new Date(),
  });
}
```

### ❌ Anti-Pattern: Cross-Guild Data Access
```typescript
// WRONG: User can access other guild's members
async function getGuildMembers(guildId: string) {
  return await memberRepository.getGuildMembers(guildId);  // No guild ownership check!
}

// CORRECT: Verify guild ownership + check if allowed
async function getGuildMembers(
  guildId: string,
  requestingUserId: string,
  isGuildOwner: boolean
) {
  if (!isGuildOwner) {
    throw new AuthorizationError('Only guild owners can view member list');
  }

  return await memberRepository.getGuildMembers(guildId);
}
```

### ❌ Anti-Pattern: Silently Failing
```typescript
// WRONG: Error swallowed
async function deleteUser(userId: string) {
  try {
    await userRepository.delete(userId);
  } catch (error) {
    // Empty catch - user never knows it failed!
  }
}

// CORRECT: Log and re-throw
async function deleteUser(userId: string) {
  try {
    await userRepository.delete(userId);
  } catch (error) {
    logger.error(`Failed to delete user ${userId}:`, error);
    throw error;  // Let caller know
  }
}
```

### ❌ Anti-Pattern: Mixing User and Guild Data
```typescript
// WRONG: User data includes guild-specific info
const response = {
  user: {
    id: user.id,
    username: user.username,
    roles: memberData.roles,          // ❌ Guild-specific!
    guildNickname: memberData.nick,   // ❌ Guild-specific!
  }
};

// CORRECT: Separate user and guild data
const response = {
  user: {
    id: user.id,
    username: user.username,
  },
  guildMember: {
    roles: memberData.roles,
    nick: memberData.nick,
  }
};
```

### ❌ Anti-Pattern: No Audit Trail
```typescript
// WRONG: Admin action with no log
async function adminBanUser(userId: string) {
  await userRepository.banUser(userId);
  // ❌ Who did this? When? Why?
}

// CORRECT: Comprehensive audit trail
async function adminBanUser(userId: string, adminId: string, reason: string) {
  await userRepository.banUser(userId);
  
  await auditRepository.logEvent({
    event_type: 'USER_BANNED',
    subject_user_id: userId,
    requesting_user_id: adminId,
    action: `Admin banned user. Reason: ${reason}`,
  });
}
```

### ⚠️ Gotcha: discord.js v13 vs v14 API Differences

**The project uses discord.js v13.17.1** due to CVE-2025-32572 in v14:

```typescript
// v13 API (✅ USE THIS)
import { MessageEmbed, Intents, Permissions } from 'discord.js';

const embed = new MessageEmbed()
  .setTitle('Title')
  .setColor('#0099ff');

const intents = [Intents.FLAGS.GUILDS, Intents.FLAGS.GUILD_MESSAGES];

const perms = Permissions.FLAGS.ADMINISTRATOR;

// v14 API (❌ DON'T USE)
import { EmbedBuilder, IntentsBitField, PermissionsBitField } from 'discord.js';
// These don't exist in v13!
```

### ⚠️ Gotcha: Database Column Names Are Snake Case
```typescript
// ❌ WRONG: Using camelCase in queries
await this.query('SELECT userId FROM users');

// ✅ CORRECT: Using snake_case
await this.query('SELECT user_id FROM users');

// Remember: Database uses snake_case, TypeScript interfaces use camelCase
```

### ⚠️ Gotcha: Soft Deletes Always Filter `deleted_at`
```typescript
// ❌ WRONG: Returns deleted records too
const users = await this.query('SELECT * FROM gdpr_user_data');

// ✅ CORRECT: Only return active records
const users = await this.query(
  'SELECT * FROM gdpr_user_data WHERE deleted_at IS NULL'
);
```

### ⚠️ Gotcha: Always JSON.stringify Arrays Before Storing
```typescript
// ❌ WRONG: Storing JavaScript array directly
await this.query(
  'INSERT INTO table (roles) VALUES ($1)',
  [['ADMIN', 'MODERATOR']]  // Array passed directly
);

// ✅ CORRECT: Convert to JSON string
await this.query(
  'INSERT INTO table (roles) VALUES ($1)',
  [JSON.stringify(['ADMIN', 'MODERATOR'])]
);
```

### ⚠️ Gotcha: Always Release Database Connections
```typescript
// ❌ WRONG: Connection leaks
const client = await this.pool.connect();
await client.query('SELECT * FROM users');
// client never released!

// ✅ CORRECT: Always release in finally
const client = await this.pool.connect();
try {
  await client.query('SELECT * FROM users');
} finally {
  client.release();  // Essential!
}
```
